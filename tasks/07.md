## тап 7: История сообщений

### Задача:

При подключении новый пользователь должен видеть последние N сообщений из общего чата (replay истории).

### Требования:

1. Сервер хранит последние 20 сообщений из общего чата (кольцевой буфер или `slice`)
2. При успешном входе (после выбора имени) новому клиенту отправляется история с заголовком `--- last N messages ---`
3. В историю попадают только публичные сообщения (не `/msg`, не системные `joined/left`)
4. История хранится в `Server`, защищена мьютексом
5. Graceful shutdown сохраняет корректность — история не теряется при конкурентном доступе

### Подсказки:

1. Структура буфера: проще всего — []string с ограничением длины:
```
history []string  // в Server
```
При добавлении нового сообщения:
```
if len(s.history) >= 20 {
    s.history = s.history[1:]  // убираем самое старое
}
s.history = append(s.history, msg)
```
2. Когда отправлять историю: сразу после того, как `userName` установлен и анонс `joined` разослан — до того как клиент начнёт получать новые сообщения.
3. Мьютекс: история читается и пишется из разных горутин — используй тот же `s.mu`. Запись при броадкасте — `Lock`, чтение при отправке истории новому клиенту — `RLock`.
